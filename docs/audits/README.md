# Audits

Point-in-time snapshots of the codebase, schema, or security posture. Each file is named with the audit type and date and is **frozen** — never edit a past audit, generate a new one instead.

## Conventions

- Filename: `<audit-type>-YYYY-MM-DD.md` (e.g. `db-review-2026-04-03.md`)
- Top-of-file note: `> Snapshot of <subject> as of <date>. Re-run /<skill> for current state.`
- Generated by the corresponding skill (`/db-review`, `/security`, `/architect`, etc.)
